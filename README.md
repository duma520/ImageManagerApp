## 第一部分：入门指南

### 1.1 什么是ImageManagerApp？

**ImageManagerApp** 是一款功能强大的图片管理软件，由独立开发者杜玛创建，致力于帮助用户高效组织、管理和检索图片资源。无论您是普通家庭用户、摄影师、设计师、研究人员还是企业团队，ImageManagerApp 都能满足您的图片管理需求。

#### 🎯 核心价值

| 用户群体 | 使用场景 | 获得价值 |
|---------|---------|---------|
| **家庭用户** | 管理家庭照片、旅游照片 | 轻松整理回忆，快速找到特定照片 |
| **摄影师** | 管理作品库、客户照片 | 按项目、客户、标签分类，提高工作效率 |
| **设计师** | 管理素材库、设计作品 | 快速检索素材，建立个人作品集 |
| **研究人员** | 管理实验图片、文献图片 | 按类别、标签整理，便于引用和查找 |
| **企业团队** | 共享图片资源库 | 多用户隔离，数据安全，统一管理 |
| **学生** | 管理学习资料图片 | 建立知识图谱，高效复习 |

#### ✨ 主要特点

- **简单易用**：粘贴即用，无需复杂设置
- **功能强大**：支持全文搜索、标签系统、分类管理
- **安全可靠**：自动备份、手动备份、恢复功能
- **多用户支持**：每个用户独立数据库，互不干扰
- **跨平台**：基于Python和Qt，支持Windows、macOS、Linux
- **性能优化**：分页加载、缩略图缓存、异步处理
- **完全免费**：开源软件，GNU Affero General Public License v3.0

即使您是第一次使用电脑，也能在30秒内学会使用ImageManagerApp的基本功能。

### 1.4 安装与配置

#### 方法一：直接运行（推荐初学者）

1. **下载程序**：访问 [GitHub Releases](https://github.com/duma520/ImageManagerApp/releases) 下载最新版本的exe文件（Windows）或可执行文件（其他系统）

2. **运行程序**：双击下载的文件即可运行，无需安装

#### 方法二：从源代码运行（推荐开发者）

1. **安装Python**：访问 [python.org](https://www.python.org/) 下载并安装Python 3.8或更高版本

2. **下载源代码**：
   ```
   git clone https://github.com/duma520/ImageManagerApp.git
   cd ImageManagerApp
   ```

3. **安装依赖**：
   ```
   pip install PySide6
   # 可选：安装更新检查功能
   pip install requests
   ```

4. **运行程序**：
   ```
   python ImageManagerApp.py
   ```

#### 首次启动配置

首次启动时，程序会自动创建以下目录结构：

```
ImageManagerApp/
├── ImageManagerApp.py      # 主程序
├── icon.ico                # 程序图标（可选）
├── backups/                # 数据库备份目录
│   └── [用户名]/            # 按用户分类的备份
├── data/                   # 用户图片数据
│   └── user_[用户名]/       # 按用户分类的图片
├── logs/                   # 日志文件
├── temp/                   # 临时文件
└── users/                  # 用户数据目录
    └── [用户名].db          # 用户数据库文件
```

## 第二部分：基础操作

### 2.1 用户管理

ImageManagerApp支持多用户环境，每个用户拥有独立的数据库和图片存储空间，互不干扰。这对于家庭多人使用、企业团队协作非常有用。

#### 👤 创建第一个用户

启动程序后，您会看到用户选择对话框：

1. 点击 **"添加用户"** 按钮
2. 输入用户名（例如："张三"、"家庭相册"、"设计部"等）
3. 点击确定
4. 新用户会出现在列表中，选中后点击 **"登录"**

#### 👥 用户管理操作

| 操作 | 步骤 | 说明 |
|-----|------|------|
| **添加用户** | 菜单栏「用户」→「添加用户」或在用户选择对话框点击「添加用户」 | 创建新用户，系统会自动创建独立的数据库 |
| **切换用户** | 菜单栏「文件」→「切换用户」 | 返回用户选择对话框，可切换到其他用户 |
| **重命名用户** | 菜单栏「用户」→「重命名当前用户」 | 修改当前登录用户的名称 |
| **删除用户** | 菜单栏「用户」→「删除用户」 | **注意：这将永久删除该用户的所有数据和备份！** |

#### 💡 实用场景举例

**家庭场景**：
- 用户"爸爸"：管理工作相关图片
- 用户"妈妈"：管理家庭照片和食谱
- 用户"孩子"：管理学习和兴趣图片

**企业场景**：
- 用户"设计部"：管理设计素材库
- 用户"市场部"：管理营销图片
- 用户"研发部"：管理技术文档图片

### 2.2 图片添加与导入

ImageManagerApp提供多种添加图片的方式，满足不同场景的需求。

#### 📋 方式一：粘贴图片（最快）

这是最常用的方式，特别适合从网页、聊天软件中快速保存图片。

**操作步骤**：
1. 在浏览器、微信、QQ、Word等任何地方找到图片
2. 右键点击图片，选择 **"复制"**（或按下 `Ctrl+C`）
3. 切换到ImageManagerApp，按下 **`Ctrl+V`**（或点击「粘贴图片」按钮）
4. 在弹出的对话框中输入图片名称
5. 点击确定

**工作原理**：
- 程序会自动将剪贴板中的图片保存到用户的专用文件夹
- 自动生成时间戳文件名，避免重名
- 将图片信息添加到数据库

#### 📁 方式二：从文件夹导入

**操作步骤**：
1. 在文件夹中找到想要导入的图片
2. 直接将图片文件拖拽到ImageManagerApp窗口
3. 输入图片名称（默认使用原文件名）
4. 点击确定

或者：
1. 点击「粘贴图片」按钮旁的下拉箭头（如果有）
2. 选择「导入图片...」
3. 在文件浏览器中选择图片文件
4. 输入图片名称，点击确定

#### 📸 方式三：截图直接保存

1. 使用系统截图工具（Windows：Win+Shift+S，macOS：Cmd+Shift+4）
2. 截取屏幕区域
3. 截图会自动保存到剪贴板
4. 切换到ImageManagerApp，按 `Ctrl+V` 粘贴

#### 🔄 自动重名处理

当导入的图片与已有图片重名时，程序会自动处理：

- 如果文件内容完全相同（基于MD5哈希），会提示"图片已存在"
- 如果文件名相同但内容不同，程序会使用哈希值命名，避免覆盖

## 第三部分：进阶功能

### 3.1 数据库备份与恢复

数据安全是图片管理的核心。ImageManagerApp提供完善的备份机制，确保您的图片数据万无一失。

#### 🔄 自动备份

程序默认每小时自动备份一次数据库，让您无需担心数据丢失。

**自动备份特点**：
- 备份文件保存在 `backups/用户名/` 目录下
- 文件名格式：`auto_YYYYMMDD_HHMMSS.db`
- 默认保留最近30个备份，自动清理旧备份
- 完全无感运行，不影响正常使用

**如何设置自动备份间隔**？
自动备份间隔固定为1小时，不可调整。如果您需要更频繁的备份，可以使用手动备份。

#### 👆 手动备份

当您要添加重要图片或进行重大操作前，建议手动备份。

**操作步骤**：
1. 菜单栏「文件」→「手动备份」
2. 程序立即创建备份
3. 状态栏显示备份完成提示

**手动备份文件名**：`manual_YYYYMMDD_HHMMSS.db`

#### 🔁 恢复数据库

当您需要恢复到之前的某个状态时，可以使用恢复功能。

**操作步骤**：
1. 菜单栏「文件」→「恢复数据库」
2. 在弹出的恢复对话框中：
   - 可以通过时间范围和备份类型筛选备份
   - 点击「预览」查看备份内容
   - 点击「恢复」选择要恢复的备份

**恢复过程**：
1. 程序会自动创建当前数据库的备份（`rollback_YYYYMMDD_HHMMSS.db`）
2. 然后从选中的备份文件恢复数据库
3. 完成后自动刷新界面

> **重要提示**：恢复操作会覆盖当前数据库！程序会自动创建回滚备份，但建议在恢复前也手动备份一次。

#### 📊 备份管理界面

恢复数据库对话框提供以下功能：

| 功能 | 说明 |
|-----|------|
| **类型筛选** | 筛选自动备份、手动备份、回滚备份 |
| **时间范围** | 选择要显示的备份时间范围 |
| **预览** | 查看备份中包含的图片数量、时间范围等信息 |
| **恢复** | 从选中的备份恢复数据库 |

### 3.2 多用户环境详解

### 5.3 多线程处理机制

#### 🧵 线程模型

```
主线程 (UI线程)
    ├── 处理用户输入
    ├── 更新界面
    └── 启动工作线程

工作线程 (Worker Threads)
    ├── ImageLoadThread  # 加载大图
    ├── ImageSaveThread  # 保存图片
    └── 数据库操作线程    # 通过连接池管理
```

#### 🔒 线程安全的数据库连接池

```python
class DatabasePool:
    def __init__(self, db_path, max_connections=5):
        self.db_path = db_path
        self.connections = {}  # 按线程ID存储连接
        self.lock = threading.Lock()
    
    def get_connection(self):
        thread_id = threading.get_ident()
        with self.lock:
            if thread_id not in self.connections:
                conn = sqlite3.connect(self.db_path)
                conn.execute("PRAGMA journal_mode=WAL")
                self.connections[thread_id] = conn
            return self.connections[thread_id]
```

#### 🖼️ 异步图片加载

```python
class ImageLoadThread(QThread):
    image_loaded = Signal(object, str)  # pixmap, file_path
    
    def run(self):
        if os.path.exists(self.file_path):
            pixmap = QPixmap(self.file_path)
            if self.target_size:
                pixmap = pixmap.scaled(
                    self.target_size.width(), 
                    self.target_size.height(), 
                    Qt.KeepAspectRatio, 
                    Qt.SmoothTransformation
                )
            self.image_loaded.emit(pixmap, self.file_path)
```

### 5.4 性能优化

#### 🚀 主要优化策略

1. **分页加载**
   - 每页200张图片
   - 避免一次性加载所有图片导致内存溢出

2. **缩略图缓存**
   ```python
   cache_key = f"thumb_{image_path}_{size}"
   pixmap = QPixmapCache.find(cache_key)
   if not pixmap:
       pixmap = QPixmap(image_path).scaled(...)
       QPixmapCache.insert(cache_key, pixmap)
   ```

3. **WAL模式**
   ```sql
   PRAGMA journal_mode=WAL;
   ```
   - 支持并发读写
   - 提高数据库性能

4. **延迟加载**
   ```python
   QTimer.singleShot(10, load)  # 延迟10ms执行，让UI先响应
   ```

5. **分批处理**
   ```python
   for i, item in enumerate(items):
       if i % 10 == 0:
           QApplication.processEvents()  # 每10个处理一次事件
   ```

## 第六部分：常见问题

### 6.1 安装与启动问题

#### ❓ Q: 双击程序没反应怎么办？

**可能原因**：
1. 缺少必要的系统运行库
2. 程序文件损坏
3. 权限不足

**解决方案**：
- **Windows用户**：尝试以管理员身份运行
- **macOS用户**：在终端中运行 `chmod +x ImageManagerApp` 添加执行权限
- **所有用户**：从源代码运行查看错误信息：
  ```bash
  python ImageManagerApp.py
  ```

#### ❓ Q: 提示 "ModuleNotFoundError: No module named 'PySide6'"

**解决方案**：
```bash
pip install PySide6
```

如果使用虚拟环境，确保已激活环境。

#### ❓ Q: 程序启动后闪退

**排查步骤**：
1. 检查是否有 `icon.ico` 文件（如果存在但损坏可能导致闪退）
2. 删除程序目录下的 `users.db` 和所有用户数据库（注意先备份！）
3. 从命令行运行查看错误信息

### 6.2 图片操作问题

#### ❓ Q: 粘贴图片没反应

**可能原因**：
1. 剪贴板中没有图片
2. 图片格式不支持
3. 权限问题

**解决方案**：
- 确认已复制图片（在画图软件中测试粘贴）
- 支持的格式：PNG, JPG, JPEG, BMP, GIF
- 检查程序目录是否有写入权限

#### ❓ Q: 图片保存失败

**错误信息**："无法保存图片文件" 或 "数据库错误"

**解决方案**：
1. 检查磁盘空间是否充足
2. 检查程序目录权限
3. 查看 `logs/` 目录下的日志文件
4. 尝试手动创建图片保存目录：
   ```
   mkdir data\user_用户名
   ```

#### ❓ Q: 搜索不到刚添加的图片

**原因**：全文搜索索引可能有延迟

**解决方案**：
1. 等待几秒钟再搜索
2. 重新启动程序
3. 检查搜索关键词是否与图片信息匹配

### 6.3 数据库问题

#### ❓ Q: 数据库被锁定

**错误信息**："database is locked"

**原因**：多线程同时访问数据库

**解决方案**：
1. 关闭程序，重新启动
2. 如果频繁出现，检查是否有多个程序实例在运行
3. 手动删除 `user_用户名.db-wal` 和 `user_用户名.db-shm` 文件（注意备份！）

#### ❓ Q: 恢复数据库失败

**错误信息**："恢复失败" 或 "无法读取备份信息"

**解决方案**：
1. 检查备份文件是否存在且完整
2. 尝试手动恢复：复制备份文件替换当前数据库文件
3. 使用SQLite工具检查备份文件完整性：
   ```sql
   PRAGMA quick_check;
   ```

#### ❓ Q: 自动备份占用太多空间

**解决方案**：
1. 修改 `BackupManager` 类的 `max_backups` 值（默认30）
2. 手动清理旧备份：删除 `backups/` 目录下不需要的文件
3. 调整备份目录位置到更大的磁盘

### 6.4 性能问题

#### ❓ Q: 图片列表加载慢

**原因**：图片数量多，或图片文件大

**解决方案**：
- 程序已经优化为分页加载，等待几秒即可
- 首次加载时会生成缩略图缓存，后续会更快
- 确保图片存储在本地硬盘（避免网络驱动器）

#### ❓ Q: 内存占用高

**原因**：打开的图片过大，或同时加载太多缩略图

**解决方案**：
1. 图片查看器关闭后会自动释放内存
2. 避免同时打开太多图片查看器窗口
3. 缩略图缓存大小受 `QPixmapCache` 限制，程序会自动管理

#### ❓ Q: CPU占用高

**原因**：后台线程在处理图片

**解决方案**：
- 这是正常现象，程序会在后台处理图片
- 等待处理完成后CPU会恢复正常
- 如果持续高占用，检查是否有线程卡死

### 6.5 故障排除

#### 🛠️ 通用排查步骤

1. **查看日志**：
   程序目录下的 `logs/app_YYYYMM.log` 文件记录了运行信息

2. **命令行运行**：
   ```bash
   python ImageManagerApp.py 2>&1 | tee error.log
   ```

3. **检查权限**：
   确保程序对以下目录有读写权限：
   - `data/`
   - `backups/`
   - `users/`

4. **重置配置**：
   删除 `QSettings` 配置（在注册表或 ~/.config 中）
